=== 코틀린 과제 채점 기준표 ===
[채점 철학]

1. **변수명/스타일 불간섭:** 개발자의 스타일을 100% 존중합니다. 변수명(`a`, `b` 등)이나 띄어쓰기로 점수를 깎지 마세요.
2. **동작 우선:** 코드가 의도한 대로 정확하게 동작하는지가 점수의 80%를 차지합니다.
3. **안전성 중시:** 예외 상황(Null, 0 등)에서도 뻗지 않는 견고한 코드인지 확인합니다.

[점수 배점표 (총 100점)]

1. ⚙️ **기능 정확성 (Functionality) - 60점**
    - **[Pass/Fail]** 모범 답안과 비교했을 때 핵심 결과값이 정확히 일치하는가? (틀리면 0점 처리)
    - 요구사항의 핵심 로직을 빠짐없이 구현했는가?
    - **주의:** 구현 방식이 모범 답안과 달라도, 결과가 맞으면 만점입니다.

2. 🛡️ **코드 견고성 및 효율성 (Quality) - 30점**
    - **불필요한 연산:** 정답 코드보다 현저히 비효율적인 로직(예: 불필요한 이중 루프)이 있는가? (-10점)
    - **하드코딩:** 입력값에 따라 바뀌어야 할 값을 고정된 숫자(예: `return 10;`)로 박아넣었는가? (-10점)
    - **데드 코드:** 실행되지 않는 의미 없는 코드 덩어리가 방치되어 있는가? (-5점)

3. 🚫 **치명적 결격 사유 (Penalty) - 10점**
    - **컴파일 불가:** 오타나 문법 오류로 실행조차 안 되는 경우 (-10점)
    - **악성 코드:** 변수명이나 주석에 **욕설, 비속어, 공격적인 언어**가 포함된 경우 (-10점)
        * (참고: `a`, `b`, `temp`, `val` 등은 절대 감점 대상이 아님)

[채점 예시 가이드]

- `val a: Int = 10; return a + b;` -> ✅ **만점** (변수명 `a`는 문제없음)
- `return 5 + 3;` -> ❌ **감점** (하드코딩: 입력변수 무시함)
- `val s: Int = a + b; val k: Int = s; return k;` -> ⚠️ **감점** (불필요한 변수 할당 남발로 가독성 저하, -2점 소폭 감점)
